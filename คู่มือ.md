# คู่มือระบบ Exchange-app

เอกสารนี้ทำไว้สำหรับบอกว่า
- หน้าไหนทำอะไร
- ถ้าจะเพิ่ม/แก้ฟีเจอร์ ต้องไปแก้ไฟล์ไหน
- ถ้าเกิดปัญหา ควรไล่เช็กจุดไหนก่อน

เอกสารที่เกี่ยวข้อง (เวอร์ชันละเอียด):
- คู่มือเริ่มจากศูนย์: [คู่มือ-เริ่มต้น-ทีมใหม่.md](คู่มือ-เริ่มต้น-ทีมใหม่.md)
- คู่มือแยกโมดูลพร้อม data flow: [คู่มือ-แยกตามโมดูล.md](คู่มือ-แยกตามโมดูล.md)

## 1) ภาพรวมโครงสร้าง

- app/ = หน้าเว็บ (Next.js App Router)
- app/api/ = API routes ฝั่ง server
- components/ = UI component ที่ใช้ซ้ำ
- lib/ = helper กลาง เช่น auth, db, security, notifications
- db/phase3_migrations.sql = migration สำคัญสำหรับ schema ใหม่

แนวคิดหลักของโปรเจกต์นี้
- หน้าเว็บคุยกับ API ใน app/api/
- API เข้าฐานข้อมูลผ่าน lib/db.js
- เรื่องสิทธิ์ใช้ session + guard จาก lib/auth.js และ lib/api-guards.js

## 2) หน้าเว็บหลักทำอะไรบ้าง

### หน้าแรก
- ไฟล์: app/page.js
- หน้าที่:
  - แสดงสินค้าอนุมัติแล้ว
  - แถบค้นหา/กรองหมวดหมู่
  - navbar หลัก (ปุ่มธีม + ปุ่มกระดิ่ง + ลิงก์หลัก)
- จุดระวัง:
  - ปุ่ม NotificationBell ตั้งใจให้มีเฉพาะหน้านี้
  - ถ้าแก้เงื่อนไข filter สินค้า ต้องระวัง approval_status และ status

### หน้ารายละเอียดสินค้า
- ไฟล์: app/items/[id]/page.js
- หน้าที่:
  - แสดงรายละเอียดสินค้า + รูปหลายรูป
  - ปุ่มเริ่มเจรจา (สร้าง request แล้วไปหน้าแชท)
  - ส่งคำร้องเรียนประกาศ
- จุดระวัง:
  - ปุ่มเจรจาเรียก API /api/requests
  - รูปและ complaint ผูกกับ session

### หน้าแชทคำขอแลก
- ไฟล์: app/chat/[requestId]/page.js
- หน้าที่:
  - แชทระหว่างเจ้าของโพสต์/ผู้ขอแลก
  - จัดการนัดหมาย
  - รีวิว/ยืนยันแลกสำเร็จ
- จุดระวัง:
  - พื้นหลังแชทใช้คลาส chat-thread-bg (โหมดสว่างจะปิด texture)
  - มี polling ทุก 3 วินาที
  - state เยอะ แก้ทีละส่วนและทดสอบทั้ง owner/requester

### หน้าคำขอทั้งหมด
- ไฟล์: app/requests/page.js
- หน้าที่:
  - รวมรายการ request ที่ user เกี่ยวข้อง
  - กดแล้วเข้าแชทราย request

### หน้าโปรไฟล์
- ไฟล์: app/profile/page.js
- หน้าที่:
  - แสดงข้อมูลผู้ใช้ สถิติ รีวิว และฟอร์มแก้ไขบางข้อมูล
- จุดระวัง:
  - มีทั้งโหมด student/admin ในไฟล์เดียว
  - แก้ UI ให้เช็กทั้งสอง role

### หน้าซัพพอร์ต
- ไฟล์: app/support/page.js
- หน้าที่:
  - เปิดเคสใหม่
  - แสดงรายการห้องซัพพอร์ต
- ไฟล์: app/support/[id]/page.js
- หน้าที่:
  - แชทกับแอดมินในห้องซัพพอร์ต
  - ปิดเคส + กดไลก์ให้แอดมิน

### หน้าแอดมิน
- โฟลเดอร์: app/admin/*
- หน้าที่:
  - ดูรายงาน, complaints, users, support, audit logs
- จุดระวัง:
  - หลายหน้าใช้ query หนัก ควรมี index ตาม migration

## 3) API สำคัญและจุดแก้

### สินค้า
- app/api/items/route.js
  - GET: ดึงสินค้า
  - POST: สร้างสินค้าใหม่
- app/api/items/[id]/route.js
  - ดู/แก้/ลบสินค้ารายตัว
- ถ้าจะเพิ่ม field ใหม่ในสินค้า:
  1) แก้ table items
  2) แก้ SELECT/INSERT/UPDATE ใน API
  3) แก้หน้า app/page.js และ app/items/[id]/page.js ให้แสดง field

### คำขอและแชท
- app/api/requests/route.js
  - สร้าง request ใหม่
- app/api/chat/[requestId]/route.js
  - GET/POST ข้อความ
- จุดระวัง:
  - สิทธิ์ owner/requester/admin ต้องคงเดิม
  - ถ้ามีรูปในแชท ต้องรองรับ image_url ขนาดใหญ่ใน DB

### นัดหมาย
- app/api/requests/[id]/meeting/route.js
  - GET/POST/PUT นัดหมาย
- พึ่งพาตาราง: exchange_meetings
- ถ้าเจอ 500 เรื่องนัดหมาย ให้เช็กทันที:
  - ตาราง exchange_meetings มีจริงไหม
  - คอลัมน์ที่ใช้ครบไหม
  - collation ของ email ตอน JOIN
  - migration รันครบหรือยัง

### แจ้งเตือน
- app/api/notifications/*
- component: components/NotificationBell.js
- policy ปัจจุบัน: แสดงกระดิ่งเฉพาะ navbar หน้าแรก

### ซัพพอร์ต
- app/api/support/conversations/route.js
- app/api/support/conversations/[id]/route.js
- app/api/support/conversations/[id]/close/route.js
- จุดระวัง:
  - ต้องคงเงื่อนไขปิดเคส (บางกรณีต้องมี admin รับเรื่องก่อน)

## 4) ธีมมืด/สว่าง

- app/layout.js
  - ตั้ง data-theme ตั้งแต่ก่อน hydration ด้วย script
- app/globals.css
  - เก็บตัวแปรธีม + compatibility layer สำหรับคลาสเดิม
  - คลาส chat-thread-bg ใช้สำหรับพื้นหลังโซนข้อความแชท

ถ้าจะเพิ่มสไตล์ที่ต้องต่างกันตามธีม
1) เพิ่มกติกาใน app/globals.css (html[data-theme="light"] ...)
2) พยายามใช้คลาสกลางแทน hardcode หลายไฟล์

## 5) ปัญหายอดฮิตและแนวทางแก้

### อาการ: นัดหมายขึ้น 500
- เช็ก:
  - app/api/requests/[id]/meeting/route.js
  - db/phase3_migrations.sql
  - schema table exchange_meetings
- แนวแก้:
  - รัน migration
  - เช็ก column + index + collation
  - ดู server log: Meeting GET/POST/PUT error

### อาการ: รูปในแชทส่งไม่ได้
- เช็ก:
  - app/api/chat/[requestId]/route.js
  - table messages.image_url
- แนวแก้:
  - ปรับเป็น MEDIUMTEXT/LONGTEXT
  - รัน migration ที่เกี่ยวกับ messages/image schema

### อาการ: โหมดสว่างอ่านยาก
- เช็ก:
  - app/globals.css (สี text-amber, text-gold-gradient)
- แนวแก้:
  - ปรับ contrast ในกติกา html[data-theme="light"]

### อาการ: พื้นหลังห้องแชทอ่านยากในโหมดสว่าง
- เช็ก:
  - คลาส chat-thread-bg ใน app/chat/[requestId]/page.js และ app/support/[id]/page.js
  - กติกา chat-thread-bg ใน app/globals.css

## 6) ถ้าจะเพิ่มเมนูใหม่ใน navbar

เริ่มที่หน้าแรกก่อน
- app/page.js

ถ้าเมนูต้องไปหลายหน้า
- แนะนำแยกเป็น component navbar กลางใน components/
- แล้วค่อยนำไปใช้ในแต่ละหน้าเพื่อลดการ copy โค้ด

## 7) เช็กลิสต์ก่อนปิดงานทุกครั้ง

1. ทดสอบ role student และ admin
2. ทดสอบ dark/light ทั้งหน้าเดิมและหน้าที่เพิ่ม
3. ตรวจ API ที่เกี่ยวข้องว่าคืน status ถูกต้อง
4. เช็ก error ใน editor และ console ฝั่ง server
5. ถ้าแตะ schema ให้แนบ migration เสมอ
