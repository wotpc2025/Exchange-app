# คู่มือเริ่มต้นสำหรับทีมใหม่ (Step-by-step)

เอกสารนี้เหมาะกับคนที่เพิ่งเข้าทีม และต้องเริ่มจากศูนย์

## Step 1: เตรียมเครื่องมือ

1. ติดตั้ง Node.js เวอร์ชัน LTS
2. เตรียม MySQL ที่เข้าถึงได้จากเครื่องที่รันแอป
3. เตรียมบัญชี Google ที่ใช้ล็อกอิน (ตามการตั้งค่า NextAuth ของทีม)
4. เปิดโปรเจกต์ใน VS Code

## Step 2: ติดตั้ง dependency

1. เปิด terminal ที่ root โปรเจกต์
2. รันคำสั่ง npm install

## Step 3: ตั้งค่า environment

สร้างไฟล์ .env.local ใน root โปรเจกต์ แล้วใส่ค่าอย่างน้อย:

- DB_HOST
- DB_USER
- DB_PASSWORD
- DB_NAME
- DB_PORT
- NEXTAUTH_SECRET หรือ AUTH_SECRET
- NEXTAUTH_URL
- GOOGLE_CLIENT_ID
- GOOGLE_CLIENT_SECRET

ไฟล์อ้างอิงการเชื่อมต่อ DB อยู่ที่ [lib/db.js](lib/db.js)
ไฟล์ auth/session อยู่ที่ [lib/auth.js](lib/auth.js) และ [app/api/auth/[...nextauth]/route.js](app/api/auth/[...nextauth]/route.js)

## Step 4: เตรียมฐานข้อมูล

1. สร้างฐานข้อมูลตามชื่อใน env
2. import schema หลักของโปรเจกต์ (ถ้ามีไฟล์ต้นทางของทีม)
3. รัน migration เพิ่มเติมที่ [db/phase3_migrations.sql](db/phase3_migrations.sql)

หมายเหตุ:
- migration นี้มี index และ schema compatibility หลายจุด
- ถ้าไม่รัน อาจเจอปัญหา meeting/chat image/support close

## Step 5: รันโปรเจกต์

1. รัน npm run dev
2. เปิด http://localhost:3000
3. ลองล็อกอิน

## Step 6: Smoke test ที่ควรผ่าน

1. หน้าแรกโหลดสินค้าได้: [app/page.js](app/page.js)
2. เปิดรายละเอียดสินค้าได้: [app/items/[id]/page.js](app/items/[id]/page.js)
3. กดทักแชทแล้วเข้าห้องได้: [app/chat/[requestId]/page.js](app/chat/[requestId]/page.js)
4. เปิด/ปิดโหมดธีมได้: [components/ThemeToggle.js](components/ThemeToggle.js)
5. เปิดห้องซัพพอร์ตได้: [app/support/page.js](app/support/page.js)

## Step 7: เข้าใจโครงสร้างไฟล์ก่อนแก้โค้ด

- หน้าเว็บ: app/*/page.js
- API: app/api/**/route.js
- UI ใช้ซ้ำ: components/*
- ธีมและ style กลาง: [app/globals.css](app/globals.css), [app/layout.js](app/layout.js)
- helper กลาง: lib/*

## Step 8: แนวทางแก้โค้ดแบบปลอดภัย

1. แก้หน้าเว็บ: เช็กว่าหน้านั้นเรียก API ไหน
2. แก้ API: เช็กว่ามีการตรวจสิทธิ์/sanitize/rate limit ครบไหม
3. แก้ DB: ถ้าแตะ schema ให้ทำ migration เพิ่มเสมอ
4. ทดสอบทั้ง role student และ admin
5. ทดสอบทั้งโหมดมืดและโหมดสว่าง

## Step 9: ถ้าจะเพิ่มฟีเจอร์ใหม่ ให้ทำตามลำดับนี้

1. ออกแบบ data model (ต้องเพิ่มคอลัมน์/ตารางไหม)
2. เพิ่ม migration SQL
3. เพิ่ม/แก้ API route
4. เชื่อมหน้า UI
5. เพิ่มข้อความ error ที่อ่านเข้าใจได้
6. ทดสอบ end-to-end
7. อัปเดตเอกสาร [คู่มือ.md](คู่มือ.md)

## Step 10: จุดที่คนใหม่มักพลาด

1. ลืมรัน migration ล่าสุด
2. แก้แต่ UI แต่ไม่แก้ API หรือ schema ตาม
3. ไม่ทดสอบ role admin ทำให้สิทธิ์รั่ว
4. ไม่ทดสอบโหมดสว่างจนอ่านยาก
5. แก้หลายหน้าแต่ไม่รวม logic เป็น component กลาง

## Step 11: Debug checklist สั้นๆ

1. ดู error ใน browser console
2. ดู response จาก API ว่า status อะไร
3. ดู server log ฝั่ง route ที่เกี่ยวข้อง
4. เช็ก query กับ schema ว่าคอลัมน์ตรงกัน
5. เช็ก session ว่ามี user/role ที่คาดไว้
